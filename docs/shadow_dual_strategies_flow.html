「每 4 秒 tick」是什么意思？
主循环每 SHADOW_DUAL_POLL_SEC=4 秒醒一次：拉 Gamma 当前 5m 市场 → 拉 YES/NO 订单簿 → 各调用一次 strat_a_tick 和 strat_b_tick → 休眠 4 秒。
不是 市场只有 4 秒；一轮 BTC 5m 约 300 秒，期间大约执行 75 次 tick（每 tick 最多各下一笔模拟单）。

2. 成交模拟 buy / walk_ask_fill（共用）

每次 BUY 都走订单簿，不是固定 best ask 满批成交
  1. 输入：想买的份数 want_shares、该边 ask ladder
  2. walk_ask_fill：从 best ask 往上逐档吃，每档量 × BOOK_TAKE_RATIO
  3. 可选滑点：成交价 × (1 + SLIPPAGE_PCT)
  4. 深度不够 → 部分成交，日志 fill 8/20sh 或 NO-FILL 吃不满
  5. 扫多档 → 日志 slip avg=… best=…
  6. 成交额 < $1 或超预算 → 放弃
  7. 更新 yes_sh / no_sh / cost / spent / fees

3. 对冲 try_hedge_light（A/B 共用）

买轻边（份数少的那边），缩小 gap
  1. 无重边 或 gap=0 → 返回 false
  2. 轻边 ask > max_light_ask → 返回 false
    A 正常对冲不限价；A-endgame 限 0.85
  3. fill = min(max_fill, gap)
  4. 边际 combined = 重边均价 + 轻边 ask > max_combined → false
  5. projected combined（整批买完后）> max_combined → false
  6. 若 require_economics → 调用 hedge_economics，不值得 → SKIP-HEDGE
  7. 现金不够（可用 emergency 额度）→ 缩小 fill 或 false
  8. buy(轻边) via walk_ask_fill

9. 策略 B — 分批对冲 b_gradual_hedge_batch

  1. fill = dynamic_batch(cash, light_ask, gap) · gap≥50 时 ×0.75
  2. projected combined > 0.98 → 每次减 5 份直到 ≤0.98 或放弃
  3. 若 gap≥50 或 重边均价≤0.30 → 跑 ECON（hedge_economics）
  4. ECON worth=false → 跳过本批
  5. try_hedge_light · max_combined=0.98 · 只对冲一批（不一次填平 gap）
